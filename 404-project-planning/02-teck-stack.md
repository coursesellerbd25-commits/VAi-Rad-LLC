# Tech Stack

## Frontend

### React
- **Purpose:** Build a modular, component-based user interface.
- **Why:** Required by the assessment and ideal for reusable components.

### TypeScript
- **Purpose:** Add static typing to JavaScript.
- **Why:** Improves code quality, catches errors early, and is required by the assessment.

### Vite
- **Purpose:** Frontend build tool and development server.
- **Why:** Extremely fast startup, hot module replacement (HMR), and excellent TypeScript support.

### Tailwind CSS
- **Purpose:** Utility-first CSS framework.
- **Why:** Enables rapid development of responsive, consistent, and modern UI.

### shadcn/ui
- **Purpose:** Reusable UI component library.
- **Why:** Provides accessible, customizable components that integrate well with Tailwind CSS.

### Zustand
- **Purpose:** Global state management.
- **Why:** Lightweight, simple API, and ideal for managing shared state such as the selected date and authenticated user.

### React Hook Form
- **Purpose:** Form state management.
- **Why:** High performance with minimal re-renders and easy integration with validation libraries.

### Zod
- **Purpose:** Schema validation.
- **Why:** Ensures form and API data are validated with strong TypeScript support.

### Axios
- **Purpose:** HTTP client.
- **Why:** Simplifies API requests, request interceptors, and centralized error handling.

### TanStack Query
- **Purpose:** Server state management.
- **Why:** Handles caching, background refetching, loading states, and synchronization with the backend.

### @dnd-kit
- **Purpose:** Drag-and-drop functionality.
- **Why:** Flexible, accessible, and well-suited for implementing a Kanban board.

### React Konva
- **Purpose:** Canvas-based drawing.
- **Why:** Simplifies drawing and managing polygons for image annotation.

### Framer Motion
- **Purpose:** UI animations.
- **Why:** Adds smooth transitions and interactions to improve user experience.

### Lucide React
- **Purpose:** Icon library.
- **Why:** Lightweight, modern, and consistent SVG icons.

## Backend

### Django
- **Purpose:** Backend web framework.
- **Why:** Required by the assessment and provides a robust architecture for REST APIs and authentication.

### Django REST Framework
- **Purpose:** Build RESTful APIs.
- **Why:** Simplifies serialization, request handling, authentication, and API development.

### SQLite
- **Purpose:** Relational database.
- **Why:** Lightweight, easy to configure, and fully sufficient for this assessment.

### Simple JWT
- **Purpose:** JWT-based authentication.
- **Why:** Secure token-based authentication for protected frontend and backend communication.
